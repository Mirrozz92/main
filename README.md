# FastSub

Платформа монетизации трафика в Telegram. Рекламодатель заказывает подписчиков
в канал/группу или запуски бота, паблишер (владелец бота) отдаёт эти задания
своим юзерам через API и забирает долю с подтверждённых подписок.

Деньги с рекламодателя списываются только когда подписка реально подтвердилась —
после hold-периода, а не в момент клика. Комиссия платформы зависит от удержания
паблишера: меньше отписок — меньше комиссия.

## Стек

Python 3.12, FastAPI, aiogram 3, SQLAlchemy 2 (async) + asyncpg, PostgreSQL 16
за pgbouncer, Redis, TaskIQ для фоновых задач. Деплой — docker-compose.

```
src/
  core/          конфиг, БД, логирование, redis
  domain/        бизнес-логика (без I/O)
  api/           публичный API для паблишеров
  bots/          aiogram-боты: main (объединённый), admin, checker
  workers/       фоновые задачи (TaskIQ)
  integrations/  CryptoBot, Telegram
  shared/        деньги, генерация id
migrations/      Alembic (0001–0010)
tests/           unit + integration
```

## Запуск локально

```bash
cp .env.example .env        # заполнить пароли, токены ботов, CryptoBot
make build && make up
make migrate
curl localhost:8000/health  # {"status":"ok"}
```

Документация публичного API — на `/docs` (человеческая страница с примерами)
и `/swagger` (голый OpenAPI).

## Тесты и линтеры

Юнит-тесты гоняются без БД. Интеграционные хотят настоящий Postgres — если его
нет, они просто скипаются, так что обычный прогон работает и так.

```bash
PYTHONPATH=. ENV=test pytest -q   # всё подряд
ruff check src tests
mypy src
```

Для интеграционных нужен Postgres и `TEST_DATABASE_URL` (по умолчанию
`postgresql+asyncpg://fastsub_test:fastsub_test@127.0.0.1:5432/fastsub_test`).
Схему строит прогон Alembic-миграций, каждый тест крутится в транзакции с
откатом. В CI базу поднимает сервис-контейнер.

CI (GitHub Actions) на каждый push и PR в `main` гоняет ruff, pytest и mypy —
всё блокирующее.

## Публичный API

Все запросы с заголовком `Authorization: Bearer <token>`. Токен паблишер берёт
в своём боте.

| Метод | Путь | Зачем |
|-------|------|-------|
| GET  | `/me` | инфо о паблишере и баланс |
| POST | `/request-op` | выдать юзеру пачку заданий |
| POST | `/check-resource` | статус одного `link_id` |
| POST | `/check-task` | статусы всех link одного задания разом |
| GET  | `/user/{id}/history` | история заданий юзера (с пагинацией) |
| GET  | `/stats` | счётчики по статусам и заработок за период |
| POST | `/webhook/configure` | настроить URL и события вебхука |

Подробности с примерами curl/Python/JS — на `/docs`.

## Бизнес-правила

- Комиссия платформы: **25%** (снижается до 20% при retention ≥ 85%)
- Минимальное пополнение рекламодателя: **500 ₽**
- Минимальная выплата паблишеру: **100 ₽**
- TTL link_id: **1 час**
- Hold-период зависит от retention паблишера: 4–12 часов
- Cold start: первые 100 подписок → hold 8 часов, комиссия 25%
- Бонус **+5%** паблишерам с retention ≥ 90%
- Коэффициенты цены за таргетинг: язык ×1.2, страна ×1.5, возраст 18+ ×2.0

## Что менялось

### Модерация ботов паблишеров (src/bots/admin/handlers/bot_moderation.py)
Новый флоу в админ-боте: список немодерированных ботов → карточка бота →
выставление тематики / возрастной / гендерной аудитории → одобрение с опциональной
заметкой. Тематика обязательна перед одобрением. Счётчик «Боты на проверке (N)»
появился в главном меню админа. Новые состояния `BotModerationStates`,
13 callback-констант, 7 клавиатур.

### Объединённый бот (src/bots/main/)
Advertiser-бот и publisher-бот объединены в один `main`-бот. Один человек может
быть одновременно рекламодателем и паблишером — всё в одном интерфейсе.
Хендлеры старых ботов переиспользуются через общий dispatcher — ничего не
переписывалось заново. Конфиг переведён на `MAIN_BOT_TOKEN`, старые токены
опциональны в `.env`.

### Таргетинг и коэффициенты цены
Добавлен модуль `src/domain/campaigns/pricing.py` с коэффициентами цены за
таргетинг. Чем точнее таргетинг — тем дороже подписчик для рекламодателя и
выше выплата паблишеру. Коэффициенты перемножаются:
- Конкретный язык: ×1.2
- Конкретная страна: ×1.5
- Возраст 18+: ×2.0
- Пол конкретный: ×1.1
- Telegram Premium: ×1.3
- Фото/ник/био: ×1.1 каждое

### Тематики ботов
Модератор выставляет боту паблишера тематику (крипто, игры, спорт...).
Рекламодатель может исключить нежелательные тематики из своего заказа.
Паблишер может исключить нежелательную рекламу из своего бота.

### Онбординг — теперь опциональный
Убрана обязательная анкета юзера при первом запросе. Паблишер сам решает
показывать ли анкету (`show_quiz` в настройках бота). Данные юзера теперь
передаются напрямую в `/request-op`: `first_name`, `username`, `language_code`.
Matchmaking фильтрует по `language_code` если рекламодатель указал языки.

### Миграции БД (0009, 0010)
- `0009` — поля модерации бота паблишера: `niche`, `age_audience`,
  `gender_audience`, `country_audience`, `is_moderated`, `moderated_at`.
- `0010` — SubGram-фичи: `daily_limit`, `track_unsubs`, `excluded_themes`,
  `language_codes`, `price_coefficient` в кампаниях; `get_links`, `show_quiz`,
  `excluded_themes` в ботах паблишера; `language_code`, `first_name` в юзерах;
  новая таблица `exclusions` для чёрных списков.

### API — новые эндпоинты
- `check-task` — статусы всего задания одним запросом (вместо N вызовов check-resource)
- `/user/{id}/history` — история заданий юзера с пагинацией
- `/stats` — статистика паблишера за период
- `/webhook/configure` — настройка вебхуков через API

### Безопасность
- HTML escape во всех ботах: `tg_username`, `full_name`, `campaign.title`
  экранируются через `html.escape()` перед вставкой в HTML-сообщения.
  Устраняет возможность HTML-инъекции через имя/ник пользователя.

### Тесты и инфраструктура
- CI на GitHub Actions: ruff + pytest + mypy на каждый push и PR.
- Юнит-тесты: шкала комиссии, split выплаты, hold, переходы статусов,
  финансовый round-trip, заявки на вывод, статистика, вебхуки.
- Интеграционные тесты: реальное движение денег, глобальный констрейнт
  «один юзер — один ресурс», concurrency-тест на `SKIP LOCKED`.

### Починки
- Отклонение заявки на вывод: `CANCELLED` → `CANCELED` (правильный член enum).
- API не стартовал: рейт-лимитер как class-instance ломал резолв `Request`.
- Логирование в checker-боте: конфликт позиционного `event` у structlog.
- Убран мёртвый код: осиротевшие хендлеры и клавиатуры.
- Добавлена зависимость `python-multipart` (без неё не работают формы).
- Краш при `verified_at = None` в финансовом сервисе.
